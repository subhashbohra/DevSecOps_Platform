# 19 — 90-Day Execution Plan

> A day-by-day plan to go from "qualified but rusty" to "interview-ready." Assumes **2 hrs/weekday, 4 hrs/weekend (~18 hrs/week)**. Adapt to your reality — the structure matters more than perfect adherence. Log everything in `18-Weekly-Tracker.md`.

**Daily default split (weekday, 2 hrs):** ~75 min coding/DSA + ~45 min system design *or* behavioral (alternate). **Weekend (4 hrs):** 1 full system design out loud + coding reps + (from Wk4) a mock.

> Files referenced: `02` (DSA), `03`/`10`/`11` (design), `09` (K8s Q&A), `12`/`13`/`14` (behavioral), `16` (referrals), `17` (mocks).

## Ongoing maintenance (after Day 90, until offers land)
- 3–5 coding problems/week (retention).
- 1 system design out loud/week.
- 1 mock/week if loops are active.
- Company guide refresh 1–2 weeks before each specific loop.
- Keep referral pipelines and public profile warm (`16`).

## Daily discipline checklist (print this)
- [ ] Did I narrate my approach + complexity *before* coding today?
- [ ] Did I test my code on a real input?
- [ ] If design day: did I *drive* and discuss tradeoffs out loud?
- [ ] Did I log it in `18-Weekly-Tracker.md`?
- [ ] Is a mock scheduled within the next 7 days (from Wk4)?

## If you fall behind (you will, some weeks)
Priority order when time is short: **(1) keep MUST-60 retention → (2) one system design/week → (3) mocks → (4) STRETCH/extras.** Protect the first three; everything else is optional. Consistency over intensity.
